### Work Machine Setup Notes

On a new machine this directory will be created empty by the server on first start. To transfer data:
1. Copy `project-memory.db` (and optionally `.db-shm`/`.db-wal`) to the same path on the target machine.
2. Copy `supervisor.toml` and `supervisor.launch.toml` — update any absolute paths inside them to match the new machine's workspace location.
3. `tray-settings.json` stores window geometry; safe to omit (will be recreated).
4. The archived DB (`project-memory_archived_2026-02-25_102039.db*`) is a backup — can be left behind.

## 5. `C:\Users\User\Project_Memory_MCP\Project-Memory-MCP\data\`

**Gitignore rule:** `data/`

The `data/` directory under the workspace root contains a local mirror/copy of workspace plan state used historically by the dashboard server. It is ignored by git and excluded from the MCP server's canonical data root (which is in `AppData\Roaming\ProjectMemory`).

This directory may be stale relative to the live database. Do not rely on it for authoritative plan state.
