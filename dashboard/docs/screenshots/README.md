# Screenshots

This folder contains screenshots for the User Guide documentation.

## Required Screenshots

To complete the documentation, capture the following screenshots:

1. **dashboard-overview.png** - Main dashboard view showing workspaces
2. **first-launch.png** - Initial dashboard state
3. **workspaces-list.png** - Workspace list with health indicators
4. **plans-list.png** - Plans page with filters
5. **plan-detail.png** - Single plan detail view with steps
6. **agents-list.png** - Agent templates list
7. **agent-detail.png** - Agent detail with deployment matrix
8. **activity-feed.png** - Real-time activity feed
9. **metrics-dashboard.png** - Metrics/analytics page
10. **vscode-sidebar.png** - VS Code extension sidebar
11. **context-menu.png** - Right-click context menu
12. **status-bar.png** - VS Code status bar item

## Capture Instructions

1. Start the dashboard: `npm run dev:all`
2. Use sample data or create test plans
3. Capture at 1280x800 resolution for consistency
4. Use PNG format with compression
5. Ensure no sensitive information is visible

## Tools

- **Windows**: Snipping Tool, ShareX
- **macOS**: Screenshot.app, CleanShot X
- **Linux**: Flameshot, GNOME Screenshot
