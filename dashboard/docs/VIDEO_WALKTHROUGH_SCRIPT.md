# Memory Observer Dashboard - Video Walkthrough Script

> A comprehensive video tutorial script for the Memory Observer Dashboard.

---

## Video Overview

| Property | Value |
|----------|-------|
| **Title** | Getting Started with Memory Observer Dashboard |
| **Duration** | ~8-10 minutes |
| **Audience** | Developers using Project Memory MCP |
| **Format** | Screen recording with voiceover |
| **Resolution** | 1920x1080 (1080p) |

---

## Pre-Recording Checklist

- [ ] Clean desktop with minimal distractions
- [ ] Dashboard running with sample data
- [ ] VS Code open with extension installed
- [ ] MCP server running (for live events)
- [ ] Microphone tested and levels set
- [ ] OBS/recording software configured
- [ ] Script printed or on second monitor

---

## Script

### Scene 1: Introduction (0:00 - 0:30)

**VISUAL:** Dashboard home page

**VOICEOVER:**
> "Welcome to the Memory Observer Dashboard - your visual command center for Project Memory MCP workflows. In this walkthrough, we'll explore how to monitor agent activities, track plan progress, and manage your AI-assisted development workflows."

**ACTION:** Pan across the dashboard interface

---

### Scene 2: Dashboard Overview (0:30 - 1:30)

**VISUAL:** Home page with workspaces

**VOICEOVER:**
> "The dashboard opens to your workspace overview. Each registered project appears here with health indicators showing activity status."

**ACTION:** Point out the health indicators

> "Green means active development, yellow indicates stale projects, and red flags blocked workflows. Let's click on a workspace to see its plans."

**ACTION:** Click on a workspace card

---

### Scene 3: Viewing Plans (1:30 - 3:00)

**VISUAL:** Workspace plans list

**VOICEOVER:**
> "Here we see all plans for this workspace. Plans are categorized by type - features, bugs, refactoring - and prioritized from low to critical."

**ACTION:** Hover over filter dropdowns

> "Use these filters to focus on what matters. Let's filter by active status to see work in progress."

**ACTION:** Apply status filter

> "Click any plan to dive into the details."

**ACTION:** Click on a plan card

---

### Scene 4: Plan Details (3:00 - 4:30)

**VISUAL:** Plan detail page

**VOICEOVER:**
> "The plan detail view is your project's story. At the top, we have the title, status, and key metadata."

**ACTION:** Point to header section

> "Below that, steps are organized by phase. Each step shows its status - pending, in progress, completed, or blocked."

**ACTION:** Scroll through steps

> "The timeline shows how agents have collaborated. Each handoff from Coordinator to Executor to Reviewer is tracked."

**ACTION:** Click on Timeline tab

> "You can also view agent session details, audit logs, and research notes from this page."

---

### Scene 5: Exporting Reports (4:30 - 5:00)

**VISUAL:** Plan detail with export dropdown

**VOICEOVER:**
> "Need to share a plan with your team? Click Export to download a Markdown report with visual diagrams, or copy the raw JSON for integrations."

**ACTION:** Click Export button, show dropdown options

> "The Markdown export includes a Mermaid timeline diagram that renders beautifully in GitHub and other platforms."

---

### Scene 6: Managing Agents (5:00 - 6:30)

**VISUAL:** Agents page

**VOICEOVER:**
> "Navigate to the Agents page to manage your agent templates. Each agent has a specific role in the workflow."

**ACTION:** Click on Agents in sidebar

> "The Coordinator analyzes requests. The Architect designs solutions. The Executor implements code. And so on."

**ACTION:** Scroll through agent cards

> "Sync status badges show you at a glance whether deployments are current. Yellow means the template was updated and workspaces need syncing."

**ACTION:** Click on an agent

> "From the detail view, you can edit the agent's instructions, deploy to specific workspaces, or pull customizations back to the template."

---

### Scene 7: Real-time Activity (6:30 - 7:30)

**VISUAL:** Activity feed page

**VOICEOVER:**
> "The Activity feed shows real-time events from your MCP server. Watch as agents work - tool invocations, step updates, handoffs."

**ACTION:** Navigate to Activity page

> "The connection indicator confirms live streaming is active. Filter by event type if you need to focus on specific activities."

**ACTION:** Show connection status, apply filter

> "This is invaluable for debugging workflows and understanding agent behavior."

---

### Scene 8: Performance Metrics (7:30 - 8:00)

**VISUAL:** Metrics page

**VOICEOVER:**
> "Finally, the Metrics page gives you the big picture. Total plans, completion rates, agent performance, category breakdown."

**ACTION:** Navigate to Metrics page

> "Use these insights to optimize your workflows and identify bottlenecks."

**ACTION:** Scroll through metrics sections

---

### Scene 9: VS Code Extension (8:00 - 9:00)

**VISUAL:** VS Code with extension

**VOICEOVER:**
> "For an integrated experience, install the VS Code extension. The dashboard lives right in your Activity Bar."

**ACTION:** Switch to VS Code, click extension icon

> "Right-click any file to add it to your active plan. Select code to include context with your step."

**ACTION:** Demonstrate context menu

> "Agent templates hot-reload when you save, keeping your AI assistants up to date."

---

### Scene 10: Conclusion (9:00 - 9:30)

**VISUAL:** Dashboard overview

**VOICEOVER:**
> "That's the Memory Observer Dashboard - complete visibility into your Project Memory workflows. Start tracking your agent activities today and take control of AI-assisted development."

**ACTION:** Show dashboard logo/branding

> "Check out the documentation for more details, and happy coding!"
